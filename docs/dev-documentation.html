Developer Documentation

  • Running Project Currently

    To run the project navigate to the project directory "/C:/Path/To/Directory/repo1. Then run python app.py for now you will need to run pip install flask and pip install peewee to get the proper dependencies

  • Obtaining the source code

    To get the source code on your machine, run git clone https://github.com/423S25/project1. We intend to implement a package manager, but for now, make sure to have Python 3 installed, with Flask and Peewee.

  • Project Structure

    The directory structure is /docs for the documentation, /src/model for the database code, and /templates for the rendered pages. app.py (the web server) is located in the root directory.

  • Building

    At this time, no building is needed. These docs will be updated when we implement a package manager.

  • Testing

    Developers should be sure to test the core functionality of the application with every contribution. At least one approving reviewer should do a code review and test each contribution.

  • Automated Building and Testing

    At this time, we don't have any automated building or testing set up. These docs will be updated when we do.

  • How to release a new version

    To release a new version of this software, update the code and send the updated code to the server.

  • Bug Tracking

    To view a list of current and resolved bugs, go to our github issues here.

  • Additional considerations

    The code for this project is stored here. Developers should provide meaningful comments when contributing source code.

  • Contribution

    This is the general expected workflow for adding a feature to the project:
    1. make a branch on github
    2. git pull on the main branch
    3. git checkout branch-name
    4. do your work, add your feature, you can save your changes to your branch
    5. push your changes (make sure you are on your branch, not main)
    6. on github, make a pull request to merge your changes into the main branch (resolve any merge conflicts)
    7. tag someone for a review
    8. once someone approves, merge it
